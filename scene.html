AI Coding 以后，更要懂底层

AI 能写 JS，
但你真的看得懂吗？

问题不是它能不能生成，而是你能不能判断。

ai-generated.js
const result = await buildPage(data)
button.onclick = () => submit(result)

// 能跑，不等于可靠
this? async?

先纠正一个误会

它不是网页速成书。

它更像一张 JavaScript 的运行地图。

不是跟着做页面
而是理解代码为什么这样运行

先把语言底座补齐

类型、变量、对象、函数。

这些不是名词，是你读懂 AI 代码的基础坐标。

value类型值到底是什么
binding变量名字绑定到哪里
shape对象数据如何组织
reuse函数行为如何复用

真正拉开差距的是机制

闭包、this、原型、异步。

bug 最常藏在这些地方，靠猜会越来越慢。

JS
Runtime
闭包 this 原型 异步

再进入浏览器现场

DOM、事件、Fetch、Storage。

页面问题，本质上是 JS 和浏览器环境一起运行。

DOM页面结构
事件用户动作
Fetch网络请求
Storage本地状态

最后补上服务端视角

Node.js 让 JS 离开浏览器。

模块、文件、HTTP、进程，决定你能不能看懂工程代码。

import模块
fs文件
requestHTTP
process进程

它适合谁？

会写页面，
但底层 bug 还靠猜的人。

从“能写”到“能判断”，这本书补的是底层理解。
